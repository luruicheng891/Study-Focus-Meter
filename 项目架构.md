# 界面流程与数据流 (主界面 → 学习状态 → 学习结束)

> 本节是理解整机运行的核心。整个交互围绕 `StateTask` 状态机展开,
> `DisplayTask` 根据 `State_GetCurrent()` 切换三块屏幕,
> `FusionTask` 持续把四路传感数据融合成一个评分供界面与统计使用。

## 1. 三块屏幕与状态的对应关系

| 状态 (LearningState_t) | 显示屏幕 | LCD 方向 | 说明 |
|------------------------|----------|----------|------|
| `ST_IDLE` | 天气仪表板 (`g_weather_screen`) | `LCD_direction(3)` | **主界面**, 时钟+天气, 控制条 "Start" |
| `ST_PICKING` | 学习界面 (`LearningScreen`) + Picker | `LCD_direction(3)` | 选择目标时长 (5~120min) |
| `ST_LEARNING` | 学习界面 (`LearningScreen`) | `LCD_direction(3)` | **学习状态**, 全系统实时监测 |
| `ST_PAUSED` | 学习界面 (秒表冻结) | `LCD_direction(3)` | 用户主动暂停 |
| `ST_AWAY` | 学习界面 (秒表冻结) | `LCD_direction(3)` | 压力消失 30s 自动进入 |
| `ST_SUMMARY` | 学习报告 (`LearningReport`) | `LCD_direction(3)` | **学习结束**, 侘寂暖灰风统计页 |

> 摄像头预览 (`g_cam_screen`) 是独立调试屏, 由 USART1 `'C'`/`'W'` 命令切换, 不在学习主流程内。

## 2. 状态转移图

```text
            GUI_START                CONFIRM_DURATION(min)
  [ST_IDLE] ───────────► [ST_PICKING] ──────────────────► [ST_LEARNING]
      ▲                       │                                  │  ▲
      │ GUI_BACK              │ GUI_END(取消)                     │  │ PRESSURE_BACK
      │                       ▼                                  ▼  │
  [ST_SUMMARY] ◄──────────────┴──────────────┐           [ST_AWAY] ─┘
      ▲   ▲                                   │              │
      │   │ GUI_END / TIME_COMPLETE           │ GUI_END      │ AWAY_TIMEOUT(5min)
      │   └───────────────────────────────────┘              │
      └──────────────────────────────────────────────────────┘

  [ST_LEARNING] ──GUI_PAUSE──► [ST_PAUSED] ──GUI_RESUME──► [ST_LEARNING]
                                    │ GUI_END
                                    ▼
                               [ST_SUMMARY]
```

触发事件来源:
- **GUI 事件** (`STATE_EVT_GUI_*`): 由 `DisplayTask` 的按钮回调 / Picker 通过 `State_PostEvent()` 投递
- **内部事件** (`PRESSURE_LOST/BACK`, `AWAY_TIMEOUT`, `TIME_COMPLETE`): 由 `StateTask` 主循环轮询压力/计时自行投递

## 3. 端到端数据流

```text
┌─────────────── 感知层 (持续运行, 不分状态) ───────────────┐
│ OV2640 → AITask         专注/分心/疲劳 概率                │
│ AHT20+BH1750+MAX9814 → Sensor_Task → xSensorDataQueue      │
│ ESP32(IMU+压力) → BLE → USART2 → Slave_RxTask (state/score/pressure) │
│ ESP-01(天气) → USART3 → Weather_RxTask                     │
└───────────────────────────────────────────────────────────┘
        │AI_GetLatestProbs   │xQueuePeek        │Slave_GetLatest
        ▼                    ▼                  ▼
┌───────────────── FusionTask (1s 周期) ─────────────────────┐
│ 视觉40% + 坐姿30% + 环境20% + 时长10% → FusionResult_t       │
│   total_score / 各子分 / 概率 / advice → 长度1队列           │
└────────────────────────────────────────────────────────────┘
        │Fusion_GetLatest                       │Slave_GetLatest
        ▼                                        ▼
┌───────────────── StateTask (200ms 周期) ───────────────────┐
│ 1) 处理事件队列 (GUI / 内部超时)                            │
│ 2) 压力监测 → AWAY / 恢复                                   │
│ 3) 倒计时检查 → TIME_COMPLETE                               │
│ 4) 累加统计: total_score / focus·distract·fatigue 占比 /    │
│    坐姿异常次数(边沿) / away·pause 次数 / 持续分心·疲劳计时   │
│    → SessionSummary_t                                      │
│ 状态切换时通过 USART2 向从机发单字符指令 A/Z/S (开始/停止/休眠)│
└────────────────────────────────────────────────────────────┘
        │State_GetCurrent / State_GetSummary / State_GetElapsedSeconds
        ▼
┌───────────────── DisplayTask (100ms 轮询) ─────────────────┐
│ 状态变化 → lv_scr_load 切屏 (主界面/学习界面/报告)          │
│ 学习中: LearningScreen_UpdateStopwatch/Sensor/Fusion/Bar    │
│ 进入 SUMMARY: State_GetSummary → LearningReport_Update      │
│ 每帧: Reminder_Poll (温和提醒) + ScreenSaver_Tick (息屏)    │
└────────────────────────────────────────────────────────────┘
```

## 4. 关键时序 (一次完整学习会话)

1. **主界面 (IDLE)**: 显示天气仪表板。用户点 "Start" → `btn_state1` 回调发 `STATE_EVT_GUI_START`。
2. **PICKING**: 点 Start 后**直接**切到学习界面 + 弹时长选择器 (此处**不**判断是否有人)。
   用户选时长 (或 5s 超时) → `STATE_EVT_GUI_CONFIRM_DURATION(min)`。
3. **确认时校验在位**: `StateTask` 收到 `CONFIRM_DURATION` 后, 先把选中的分钟数写入 `s_target_minutes`,
   再读 `Slave_GetLatest()`: 压力 > `STATE_PRESSURE_THRESHOLD(5.0)` 或无 BLE 数据(调试模式) → 进入 LEARNING;
   没人则 `State_SetMessage("Please sit on chair")` 并**退回 IDLE** (坐下后可重新点 Start)。
4. **LEARNING**: 进入时向从机发单字符 `A` (开始采集), 启动秒表 (`s_learning_resume_tick`),
   `target_seconds = min*60`。此后:
   - `FusionTask` 每秒产出评分, 学习界面环形 + DBG 面板实时刷新;
   - `StateTask` 每 200ms 累加 `SessionSummary` 统计;
   - `Reminder` 检测持续分心/疲劳/坐姿异常/45min 一轮, 浮层温和提示;
   - 压力消失 ≥30s → `ST_AWAY` (秒表冻结, away_count++); 压力恢复 → 回 `ST_LEARNING` (重发 `A`);
   - 点 "Pause" → `ST_PAUSED` (秒表冻结, pause_count++, 发 `Z` 停止采集; 继续时回 LEARNING 重发 `A`)。
5. **学习结束 (SUMMARY)** 三种触发:
   - 手动点 "End" → `auto_ended=0`;
   - 倒计时达成 (elapsed ≥ target) → `TIME_COMPLETE`, `auto_ended=2`;
   - AWAY 持续 5min → `AWAY_TIMEOUT`, `auto_ended=1`。
   进入 SUMMARY: 发 `Z` (停止采集), 计算总时长/平均分/占比, `DisplayTask` 读 `State_GetSummary()`
   → `LearningReport_Update()` → 切到学习报告屏。
6. **返回主页**: 报告页点 "Back" → `STATE_EVT_GUI_BACK` → `ST_IDLE`, 清空会话统计, 切回天气仪表板。

> **主机 → 从机控制指令 (USART2, 单字符)**: `A`=开始采集/上报, `Z`=停止采集/上报, `S`=休眠 (预留, 经 `Slave_Sleep()` 触发)。
> 由业务层 `Slave_SendCmd()` 封装, 底层 `UART_ESP32_SendByte()` 阻塞发送 (TX 与 RX DMA 互不干扰)。

## 5. 关键数据结构 (流转载体)

```c
/* FusionTask 产出, 1s 周期, 长度1队列 */
typedef struct {
    uint8_t total_score;                  /* 加权总分 0..100 */
    uint8_t vision_score, posture_score, env_score, duration_score;
    uint8_t env_temp_score, env_humi_score, env_lux_score, env_noise_score;
    float   vision_prob_focus, vision_prob_distract, vision_prob_fatigue;
    char    posture_state[16];  float pressure;
    int32_t temperature_c100;   uint32_t humidity_x100, lux_x100;
    uint16_t sound_pct;         uint32_t study_minutes;
    uint32_t flags;             char advice[64];
    uint32_t timestamp, seq;
} FusionResult_t;

/* StateTask 结束时填好, GUI 读取 → LearningReport */
typedef struct {
    uint32_t total_seconds;          /* 实际学习时长 */
    uint16_t target_seconds;         /* 目标时长 (0=无目标) */
    uint8_t  avg_total_score;        /* 平均总分 */
    uint8_t  focus_pct, distract_pct, fatigue_pct;
    uint8_t  posture_abnormal_count; /* 坐姿异常次数 (边沿计数) */
    uint32_t away_count, pause_count;
    uint8_t  auto_ended;             /* 0=手动 1=5min离开 2=时长达成 */
} SessionSummary_t;   /* == LearningReport 的 LRSessionData_t */
```

# 一、项目定位

本项目是一个：

```text
基于 STM32H7 的多模态边缘AI学习状态感知系统
```

系统融合：

- AI视觉分析
- 姿态检测
- 环境监测
- 无线低功耗IoT节点
- FreeRTOS实时系统
- LVGL图形界面

用于：

```text
实时分析用户学习状态与专注度
```

并提供：

- 坐姿提醒
- 环境优化建议
- 疲劳检测
- 学习时长统计

# 七、FreeRTOS系统架构

系统基于：

```text
FreeRTOS
```

进行任务调度。

------

# 任务划分

| 任务 (实际文件) | 功能 |
| --------------- | ---- |
| LVGL_Task | 调用 lv_timer_handler() 驱动 LVGL |
| Camera_task | OV2640 摄像头采集 + canvas |
| AI_InferTask | CubeAI 推理 (专注/分心/疲劳) |
| Sensor_Task | AHT20 + BH1750 + MAX9814 环境采集 |
| Slave_RxTask | ESP32 BLE 透传 (姿态/压力) 接收解析 |
| Weather_RxTask | ESP-01 WiFi 天气接收解析 |
| Fusion_Task | 多模态融合评分 (1s) |
| State_Task | 学习会话状态机 (200ms) |
| DisplayTask | LVGL 三屏切换 + 实时刷新 + Reminder/ScreenSaver |
| Touch_Task | XPT2046 触摸 + LVGL indev |

> 详细栈大小/优先级见 `开发笔记.md` 与 `.kiro/steering/AGENTS.md` 的任务表。

# 十、项目设计原则

# 核心思想

## 1️⃣ 先闭环再扩展

优先实现：

```text
摄像头
↓
CubeAI
↓
专注识别
↓
GUI显示
```

再逐步增加：

- 环境监测
- BLE椅子节点
- 多模态融合

------

## 2️⃣ 稳定性优先

比赛中：

```text
稳定演示
>
复杂功能
```

------

## 3️⃣ 产品化优先

目标不是：

```text
传感器拼接实验
```

而是：

```text
真正智能学习终端
```
